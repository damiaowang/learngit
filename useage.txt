git的使用方法
注意点：git管理的是修改，也就是说，每次修改了文件中的数据，必须要将它加入到暂存区中，所做的
修改才能被提交
例：一个文件先修改了一次，将其加入暂存区；再进行修改，然后提交，那么实际只是提交了第一次修改
第二次的修改由于没有放入暂存区，所以并未提交。虽然是对同一个文件进行修改，但由于git管理的是
每一次的修改，而不是整个文件，所以只有加入暂存区的才会被提交

git checkout -- readme.txt
把对工作区中的readme.txt的修改全部撤销：有两种情况
1、readme.txt自修改后从未放到过暂存区，撤销回到版本库
2、readme.txt已添加到暂存区，又作了修改，撤销回到添加到暂存区后的状态

返回点是最近一次git commit或git add时的状态

使用git reset HEAD <FILE>撤销暂存区的修改，重新放加工作区
git reset HEAD readme.txt

git reset可回退版本，也可把暂存区的修改回退到工作区，HEAD表示最新的版本
